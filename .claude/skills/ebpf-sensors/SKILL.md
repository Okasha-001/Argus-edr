# eBPF sensors (ARGUS)

The kernel half of ARGUS. Programs here run inside the verifier's straitjacket and
must never be able to crash the kernel. They **only collect and forward** — no
policy, no interpretation.

## The ABI is one contract in two files

`struct event` in `bpf/common.h` and `wireEvent`/offsets in
`internal/decode/wire.go` describe the same bytes. They must agree exactly:

- Field order is chosen to avoid padding (8-byte fields, then 4, then 2, then
  byte arrays). Keep it that way; don't insert a `__u32` between the byte arrays.
- `MAX_ARGS_LEN` is a power of two so the argv loop can mask the write offset.
- Change the struct → update `common.h`, `wire.go` offsets, `WireSize`, the
  `EventType` enum in both languages, and `wire_test.go`, in one commit.

## Verifier survival kit

- **Stack is 512 bytes.** Anything larger (the 848-byte `struct event`, the argv
  buffer) lives in a per-CPU array map, fetched with `bpf_map_lookup_elem` and
  zeroed before use. Never put a big struct on the stack.
- **Loops must be bounded.** Use `#pragma unroll` with a fixed bound
  (`MAX_ARGV_COUNT`) and a mask on any index you write through:
  `off &= (MAX_ARGS_LEN - 1)`.
- **Reads must be safe.** User memory: `bpf_probe_read_user[_str]`. Kernel
  memory and tracepoint `__data_loc` fields: `bpf_probe_read_kernel[_str]`.
  Pointer-chase kernel structs with `BPF_CORE_READ` so CO-RE relocates offsets.
- You can compile (`make bpf`) without privileges, but **loading runs the
  verifier and needs root/CAP_BPF**. A clean compile is necessary, not
  sufficient — test loading on a VM.

## CO-RE / portability

- Include `vmlinux.h` (generated by `make vmlinux` from the host's BTF). Field
  names come from *that kernel's* BTF — they can differ between versions (e.g.
  `child_comm` is a `__data_loc` field on newer kernels). Read with `BPF_CORE_READ`
  and check `vmlinux.h` when a field is missing.
- Don't hardcode struct offsets; let CO-RE do it.

## Adding a program

1. Pick the most stable hook available: tracepoint > fentry/fexit > kprobe.
   Tracepoints have a stable ABI; kprobes are tied to internal symbols.
2. Build the event via `new_event(TYPE)` then `emit(e)`.
3. Filter early and cheaply in the kernel to spare the ring buffer (e.g. openat
   forwards only writes/creates).
4. Register the program in `sensorAttachments` (`internal/bpfloader`).

## Enforcement (`edr_lsm.bpf.c`)

LSM programs can return `-EPERM` to deny. This is gated by the userspace-set
`enforce_config` map: `0` off, `1` dry-run (record only), `2` enforce. Default is
off. Anything that can block must be reversible and respect the allowlist. See
`docs/SAFETY.md` before changing it.
